# TradingJournal Design System

> 상태: **Normative UI contract**
> 기준 소스: `app/static/app.css`, `app/templates/app.html`, `app/templates/login.html`
> 레이아웃 계약: `DASH_LAYOUT_REV = 7` (2026-07-15)
> 적용 범위: 로그인, 앱 셸, 대시보드, 매매일지 도크, 거래, 인사이트, 설정

이 문서는 TradingJournal의 시각 언어와 인터랙션 규칙을 다음 작업자가 일관되게 재사용하도록 만든 정본이다. 실제 렌더링 값은 위 세 런타임 파일이 실행 정본이며, 이 문서는 의도와 허용 범위를 규정한다. 토큰이나 레이아웃 계약을 의도적으로 바꿀 때는 런타임 코드와 이 문서를 같은 변경 단위에서 갱신한다.

## 1. 제품 경험 원칙

TradingJournal은 장식형 대시보드가 아니라 **판단을 기록하고 검증하는 거래 터미널**이다.

1. **Exchange-terminal density** — 넓은 화면을 모두 쓰되 정보 위계를 잃지 않는다.
2. **Borderless at rest** — 패널은 평시 한 장의 작업면처럼 보이고, hover·focus·편집 중에만 경계를 드러낸다.
3. **One scroll owner** — 데스크톱의 시장 컬럼, 개인 브리프, 매매일지 도크가 각자 한 개의 스크롤만 가진다. 패널 내부 중첩 스크롤은 금지한다.
4. **Honest data** — 실데이터, 추정치, 설명용 구간, 미연동 상태를 명확히 구분한다. 데이터가 없으면 그럴듯한 숫자를 만들지 않는다.
5. **Action over decoration** — 색과 강조는 다음 행동, 손익, 위험, 데이터 상태를 설명할 때만 사용한다.
6. **Information is not removed** — 반응형이나 밀도 모드에서 정보를 영구 삭제하지 않는다. 요약·균형·상세는 매매일지 카드에만 적용하며 대시보드 레이아웃을 바꾸지 않는다.

### 시그니처

- 상단 수평 내비게이션
- 전체 폭을 쓰는 12열 시장 워크스페이스
- 좁고 고정적인 우측 개인 브리프
- 하단 고정 매매일지 도크
- Pretendard 기반 한글 편집 체계와 JetBrains Mono 숫자
- 단일 acid-lime 주요 CTA

## 2. 금지 패턴

- 유리 질감, ambient glow, 과도한 그림자, 장식용 그라디언트
- 모든 `div`에 테두리를 두르는 카드 중첩
- 동일 화면에서 여러 개의 acid-lime CTA
- 손익과 무관한 장식용 초록·빨강
- 패널마다 세로 스크롤을 부여하는 구조
- 데이터가 없는 상태에서 샘플 수치·가짜 이벤트·가짜 청산맵 표시
- pill을 기본 컨테이너나 탭에 남용
- `word-break: break-all`로 한글을 글자 단위 분리
- 본문·숫자·라벨에 임의 폰트 크기 추가
- 요약·균형·상세 선택으로 시장 패널 크기나 패딩 변경
- 모바일에서 SL, TP1–TP3, 진입 근거, 계획, 메모를 상세 모드에서도 숨김

### 3.2 서체

| 역할 | 토큰 / 값 | 규칙 |
|---|---|---|
| 한글·UI | `--f-body`, Pretendard | 제목, 본문, 버튼, 한국어 라벨 |
| 디스플레이 | `--f-display`, Pretendard | 페이지 제목, 핵심 수치 |
| 숫자·시각·심볼 | `--f-mono`, JetBrains Mono → Pretendard | 가격, 손익, R, 타임스탬프, 짧은 영문 라벨 |

- 숫자는 `font-variant-numeric: tabular-nums`를 기본으로 한다.
- 한글 폴백은 `Malgun Gothic`, 영문 숫자 폴백은 `Cascadia Code`, `Consolas`다.
- 운영 UI에 브라우저 기본 serif, 굴림, 바탕을 직접 지정하지 않는다.

### 3.3 타입 스케일

| 토큰 | 크기 | 역할 |
|---|---:|---|
| `--fs-micro` | 11px | 짧은 메타·코드·범례 |
| `--fs-cap` | 11.5px | 표 헤더·eyebrow 보조 |
| `--fs-xs` | 12.5px | 캡션·보조 숫자 |
| `--fs-sm` | 13.5px | 패널 제목·보조 본문 |
| `--fs-base` | 14px | 기본 본문 |
| `--fs-md` | 15px | 강조 본문·도크 제목 |
| `--fs-lg` | 19px | 핵심 패널 수치 |
| `--fs-xl` | 24px | 큰 제목 |

본문 기본은 `14px / 1.55 / -0.01em`이다. 패널 제목은 `13.5px / 620`, 페이지 제목은 `28px / 600 / 1.05`를 기준으로 한다. 데이터의 중요도는 색보다 크기·무게·정렬로 먼저 표현한다.

### 3.4 간격·모서리·그림자

| 종류 | 값 |
|---|---|
| spacing | `4, 8, 12, 16, 20, 24, 32, 40px` (`--s1`…`--s10`) |
| 페이지 좌우 | `clamp(16px, 1.1vw, 24px)` (`--page-pad`) |
| 대시보드 패널 간격 | `12px` (`--dash-gap`) |
| 패널 내부 | `12px 12px` (`--panel-pad-y/x`) |
| 중첩 정보 블록 | `10px 10px` (`--nested-pad-y/x`) |
| 패널 shell | `10px` (`--panel-shell-pad`) |
| 섹션 사이 | `16px` (`--section-space`) |
| 버튼·입력 | `4px` radius |
| 배지 | `2px` radius |
| 카드·패널 | `2–3px` radius |
| pill | 상태 수치·작은 배지에만 `9999px` |

기본 패널은 shadow가 없다. 모달이나 드래그 ghost처럼 실제로 떠 있는 레이어만 `--card-elev`을 사용할 수 있다. 로그인 카드는 `--inset`만 허용한다.

### 3.5 움직임

- hover·focus·handle 노출: `120–140ms`
- 상대강도 marker 등 데이터 위치 전환: 최대 `400ms`
- 레이아웃 drag는 실제 포인터를 따라가는 ghost와 drop placeholder를 함께 표시한다.
- `prefers-reduced-motion: reduce`에서 장식성 애니메이션을 제거한다.
- 상태 갱신 애니메이션으로 수치 의미를 과장하지 않는다.

## 4. 앱 셸과 레이아웃 계약

### 4.1 앱 셸

- 상단 내비게이션 높이: 최소 `56px`, sticky.
- 콘텐츠는 `max-width`로 제한하지 않고 가로 화면을 모두 사용한다.
- 페이지 좌우 여백은 `--page-pad`만 사용한다.
- 상단 검색·거래 불러오기·새로고침은 같은 높이와 radius 체계를 유지한다.

### 4.2 데스크톱 대시보드

```text
┌─────────────────────────────── top navigation ───────────────────────────────┐
│ workspace controls · integrity state · layout lock                          │
├────────────────── market workspace ───────────────┬─ personal brief ────────┤
│ 12-column movable/resizable panels                │ one vertical column     │
│ owns one vertical scroll                          │ owns one vertical scroll│
├──────────────────────── fixed journal dock ──────────────────────────────────┤
│ summary / balanced / detail · sort · horizontal/vertical content as defined │
└───────────────────────────────────────────────────────────────────────────────┘
```

- 기본 열: `minmax(0,1fr) 5px minmax(360px,var(--brief-w))`.
- 개인 브리프 폭: 기본 `clamp(400px,27vw,520px)`, 1920px 이상 `clamp(420px,24vw,560px)`, 1440px 이하 `clamp(360px,28vw,420px)`.
- 중앙 구분선으로 개인 브리프 폭을 `360–760px` 안에서 조절한다.
- `1280px` 이하에서는 `오늘 할 일 → 보유 포지션 → 시장 정보 → 행동·규율·지표` 순서로 한 컬럼에 쌓고 패널 편집을 끈다. 긴급 패널은 DOM reparent로 올리며 `display:contents`는 사용하지 않는다. 좁은 화면의 자동 배치가 데스크톱 저장 순서를 덮어쓰면 안 된다.
- 시장 컬럼은 12열, `grid-auto-rows:8px`, `grid-auto-flow:row`, `12px` gap을 쓴다. dense 자동 재배치는 금지한다.
- 같은 시각 행의 패널은 각 패널의 자연 높이와 사용자 높이 중 큰 값을 계산한 뒤 행의 최대 row span으로 맞춘다. 사용자 순서가 12열을 넘기면 그 지점부터 새 행으로 계산한다.
- 시장 컬럼이 2000px 이상이면 OI·상대강도 내부 목록을 2열, 자금 흐름을 4열로 재배치한다. 패널 전체를 임의의 좁은 `max-width`로 제한하지 않는다.

### 4.3 기본 패널 배치

`DASH_LAYOUT_REV = 7`의 초기값이다. 사용자의 저장된 배치가 있으면 순서·span·높이를 보존하고 새 패널만 canonical order 뒤에 추가한다.

| 순서 | 시장 패널 | 기본 span | 최소 span | 기본 높이 | 최소 높이 |
|---:|---|---:|---:|---:|---:|
| 1 | 주요 일정 | 12 | 12 | 84 | 72 |
| 2 | 시장 컨텍스트 | 12 | 12 | 128 | 104 |
| 3 | 시장 레짐 | 4 | 3 | 320 | 280 |
| 4 | 미결제약정·포지셔닝 | 8 | 4 | 320 | 220 |
| 5 | BTC 대비 상대강도 | 8 | 4 | 260 | 180 |
| 6 | 자금 흐름 | 4 | 3 | 260 | 180 |
| 7 | 예측형 청산맵 | 12 | 4 | 180 | 140 |

개인 브리프 기본 순서는 `오늘 할 일 → 보유 포지션 → 행동 인사이트 → 규율·습관 비용 → 핵심 지표`다. 보유 포지션 기본 높이는 `160px`다. 거래가 없으면 `보유 포지션 → 시작하기`를 표시한다.

패널 span 정지점은 `3, 4, 6, 8, 9, 12`다. 높이는 `20px` 단위로 스냅하고 최대 높이는 `1200px`다. 시장 레짐과 OI 사분면은 서로 다른 의미의 카드로 유지한다. OI 사분면은 가격 변화와 OI 변화의 맥락이므로 `미결제약정·포지셔닝` 안에 둔다.

### 4.4 매매일지 도크

- 화면 하단 fixed, 헤더 `46px`, 데스크톱 기본 본문 높이 `220px`, 최대 `70vh`.
- 가운데 grip으로 높이를 조절하고 클릭으로 접고 편다.
- 도크 전체가 단일 스크롤 소유자다. 개별 거래 카드는 내부 스크롤을 만들지 않는다.
- 정렬 기본값은 최신순이다.

| 모드 | 최소 카드 폭 | 최소 높이 | 노출 원칙 |
|---|---:|---:|---|
| 요약 | 280px | 108px | 체결·방향·손익·핵심 상태 |
| 균형 | 420px | 168px | 계획·근거·SL/TP 핵심, 긴 텍스트 2줄 |
| 상세 | 560px | 184px | SL, TP1–TP3, 전략, 근거, 계획, 메모, 복기 전체 |

모바일 도크 최소 높이는 요약 `120px`, 균형 `168px`, 상세 `260px`이다. 저장 높이가 모드 최소값보다 작으면 해당 모드 기본값으로 복원하며 상세 모드는 정보 항목을 생략하지 않는다. 주요 터치 제어는 최소 `36px`이다.

## 5. 패널과 컴포넌트

### 5.1 대시보드 패널

- 구조: `30px chrome → content → resize handle`.
- 평시 border는 transparent, hover·focus-within에서 `--line-2`.
- 첫 방문은 레이아웃 잠금 상태다. 잠금 상태의 행동명은 `레이아웃 편집`, 편집 상태는 `편집 완료`다.
- drag·resize 중 `Escape` 또는 pointer cancel은 시작 순서·span·높이로 복원하고 localStorage에 저장하지 않는다.
- 보유 포지션은 기본 폭에서 6개 의미 열을 유지하되, 패널 콘텐츠 폭 `460px` 이하에서는 `심볼/방향/손익 → 가격 → 주문/계획 → 행동`의 4행 구조로 바꾼다. 정보 삭제와 기본 가로 스크롤은 금지한다.
- 제목과 본문 배경은 동일한 `--card`; 제목 아래 한 줄 hairline으로만 구분한다.
- chrome 제목: `--fs-sm`, weight `620`, 한 줄 말줄임.
- 조작 잠금 시 drag·resize handle은 보이지 않고 pointer event도 받지 않는다.
- 편집 시 제목 바를 잡아 이동하고 우하단 모서리로 크기를 조절한다.
- drag ghost는 포인터를 따라가며, 실제 drop 위치에는 placeholder를 표시한다.

### 5.2 카드·표·지표

- 내부 카드에는 새 border를 추가하기 전에 배경·간격·타입 위계로 구분 가능한지 먼저 판단한다.
- 표 헤더는 `--fs-cap / 600 / --pebble`; 숫자는 우측 정렬과 tabular nums.
- 행 hover는 `--card-2`, 선택은 indigo 계열로 표현한다.
- 핵심 지표는 좁은 개인 브리프에서 2열을 기본으로 하며 불필요한 높이를 만들지 않는다.
- 빈 상태는 무엇이 비었는지와 다음 행동을 한 문장씩 제공한다.

### 5.3 버튼·탭·배지

- 기본 버튼: transparent, `1px --line-2`, radius `4px`.
- primary: acid-lime 배경, 검은 텍스트. 화면당 하나.
- danger: red outline/soft fill; 삭제·연결 해제처럼 위험한 행동에만 사용.
- 탭은 pill이 아니라 하단 2px indicator 또는 작은 segmented control을 사용한다.
- 상태 배지는 radius `2px`; 장식용 배지 금지.
- Google 로그인 버튼은 Google의 흰색 브랜드 버튼 규칙을 예외로 허용한다.

## 6. 반응형·오버플로우 계약

### 6.1 컨테이너 기준

- 시장 컬럼 `≤760px`: 시장 컨텍스트 2열, 상대강도·자금 흐름 내부 1열.
- 시장 컬럼 `≤520px`: 시장 컨텍스트 1열, 섹터·세부 정보는 1열 또는 의미가 유지되는 2열.
- 개인 브리프 `≤360px`: 지표와 카드 grid 1열.
- 개별 패널 `≤1100px`: 시장 컨텍스트 2×2.
- 개별 패널 `≤700px`: OI·레짐·상대강도 복합 영역을 1열로 재배치.
- 개별 패널 `≤520px`: 표·사분면·메타 영역을 세로 흐름으로 전환.

### 6.2 뷰포트 기준

- `≤1280px`: 메인 2컬럼을 1컬럼으로 쌓고 패널 내부 overflow를 해제한다.
- `≤560px`: drag·resize handle을 숨기고 패널 padding/gap을 `14px`로 높인다. 매매일지는 1열이다.
- 검수 기준 뷰포트: `3440×1392`, `1920×1080`, `1440×900`, `1280×800`, `390×844`.

### 6.3 텍스트와 스크롤

- 한글은 `word-break: keep-all`; 긴 URL·심볼·원장 ID만 `overflow-wrap:anywhere`.
- 가격·손익·시각은 가능한 한 한 줄로 유지하고, 열 폭이 부족하면 정보 묶음 단위로 다음 줄로 보낸다.
- 중요한 라벨을 글자 단위로 쪼개지 않는다.
- 데스크톱 스크롤바는 `6px`, transparent track, `--iron` thumb, hover `--steel`.
- horizontal overflow가 의미 있는 타임라인·이벤트 strip은 가로 스크롤을 허용한다.
- 패널 내부 중첩 세로 스크롤은 예외 없이 제거하고 상위 컬럼에 위임한다.

## 7. 데이터·카피 규칙

### 7.1 데이터 정직성

- 모든 외부 데이터는 공급자와 기준 시각을 표시한다.
- `실데이터`, `추정`, `경험적 백분위`, `미연동`, `갱신 지연`을 같은 시각 상태로 섞지 않는다.
- OI 70·90백분위는 상위 30%·10% 설명 구간이다. 스퀴즈 확률이나 수익률로 검증된 임계값처럼 표현하지 않는다.
- REGIME v2.1은 canonical 15m 가격 2축이며 OI/funding을 레짐 label에 섞지 않는다. 1h·4h는 설명용 확정 context다.
- BTC 대비 상대강도는 단순 등락률이 아니라 BTC 대비 초과수익과 베타 조정값을 구분해 표기한다.

### 7.2 한국어 제품 문체

- 번역투와 AI식 설명을 피한다. 짧은 명사형 라벨 + 명확한 행동 문장을 쓴다.
- `안전/위험`처럼 결과를 단정하기보다 `보통/높음/매우 높음`, `확인 필요`, `평가 보류`를 우선한다.
- 방법론 설명은 본문을 밀어내지 않도록 `?` 도움말 rollover에 넣는다.
- 오류 메시지는 `무엇이 다름 → 앱이 무엇을 멈춤/보존함 → 사용자가 할 일` 순서다.
- 손익 관련 용어는 `평균 손익비(R)`로 통일한다.

## 8. 인터랙션·접근성

- 모든 버튼·탭·카드는 키보드로 접근 가능해야 한다.
- `:focus-visible`은 `2px --focus`, offset `2px`를 사용한다.
- drag는 방향키 이동, resize는 방향키 20px·span stop 이동을 함께 지원한다.
- 레이아웃 잠금 상태와 `aria-pressed`, 도크 확장 상태와 `aria-expanded`를 동기화한다.
- 아이콘-only 버튼은 한국어 `aria-label`을 가진다.
- 색만으로 상태를 전달하지 않고 텍스트·아이콘·위치를 병행한다.
- destructive action은 별도 확인 절차를 유지한다.

## 9. 새 패널 추가 계약

새 대시보드 패널을 추가할 때 다음을 한 변경 단위로 처리한다.

1. `DASH_PANEL_LABELS`에 정식 한국어 제목 추가.
2. `DASH_PANEL_GEOMETRY`에 `height`, `minHeight` 추가.
3. `renderDashboard()`에 기본 span·min span·초기 순서 지정.
4. 저장 레이아웃 구조가 달라지면 `DASH_LAYOUT_REV`와 migration의 canonical order 갱신.
5. 시장 데이터면 `/api/market` 등 기존 payload 재사용 여부를 먼저 검토하고, 새 호출에는 캐시·기준 시각·오류 상태 추가.
6. panel 자체가 아닌 상위 컬럼이 스크롤을 소유하는지 확인.
7. `1100 / 760 / 700 / 520 / 360px` container query에서 줄바꿈과 표 붕괴 확인.
8. 데이터 없음·지연·오류·부분 응답 상태를 각각 확인.
9. drag, resize, reset, lock, localStorage migration을 확인.
10. 위 다섯 검수 뷰포트와 키보드·reduced motion을 확인.

## 11. 변경 기록 규칙

디자인 시스템 변경 커밋에는 아래를 남긴다.

- 바뀐 토큰 또는 계약
- 왜 기존 계약으로 해결할 수 없었는지
- 영향을 받는 컴포넌트와 뷰포트
- 전후 스크린샷 또는 측정값
- `DESIGN.md` 동시 갱신 여부

일회성 CSS override로 계약을 우회하지 않는다. 예외가 필요하면 이유와 제거 조건을 코드 주석과 변경 기록에 함께 남긴다.
